# FPS Shooter

Проект на **Unity** — шутер от первого лица с командами, раундами и ботами. Ниже — как открыть проект, что значит «запустить» его и как собрать игровую сцену, если готовой сцены ещё нет.

---

## С чего начать (очень кратко)

1. Установите **Unity Hub** с [сайта Unity](https://unity.com/download).
2. В Hub установите редактор той же версии, что у проекта (см. раздел [Версия Unity](#версия-unity)).
3. В Hub: **Projects → Add → Add project from disk** и укажите папку `FPS Shooter` (ту, где лежит эта папка вместе с `Assets`, `Packages`, `ProjectSettings`).
4. Откройте проект двойным щелчком. Первый раз Unity подтянет пакеты — подождите окончания импорта (нижняя полоса прогресса).
5. Нажмите **Play** (треугольник) вверху редактора — это и есть «запуск» в редакторе.

**Важно:** в `SampleScene` пока может не быть пола, игрока, ботов и менеджера раундов — код уже в проекте, но сцену нужно собрать по инструкции ниже, иначе при Play будет пусто или ничего не будет работать как игра.

---

## Версия Unity

В репозитории зафиксирована версия в `ProjectSettings/ProjectVersion.txt`. Сейчас это:

- **Unity 6000.3.11f1** (ветка Unity 6).

Как поставить именно её:

1. Откройте **Unity Hub → Installs**.
2. **Install editor** → выберите версию **6000.3.11f1** (или максимально близкую в той же линейке 6000.3.x, если точной сборки в Hub нет).

Если версия сильно отличается (другая major-версия), Unity может попросить обновить проект или конвертировать ассеты — лучше держаться указанной версии.

## Unity Hub: куда нажимать (подробно)

Интерфейс Hub чаще на **английском** — ниже те подписи, которые вы увидите на кнопках.

### Установить редактор Unity

1. Запустите **Unity Hub**.
2. Слева выберите вкладку **Installs** (установки).
3. Справа сверху нажмите **Install Editor** (или **Install**).
4. Откройте вкладку с **архивными** версиями, если нужной сборки нет в списке по умолчанию: **Installs → Gear (шестерёнка) → Archive** / или выберите версию из списка **6000.3.11f1**.
5. В мастере установки отметьте модуль **Microsoft Visual Studio Community** (или **Visual Studio**) под Windows — так проще отлаживать C#-скрипты. Остальное можно оставить по умолчанию.
6. Дождитесь окончания загрузки и установки.

### Добавить этот проект в список

1. В Hub слева вкладка **Projects**.
2. Нажмите стрелку у **Add** (справа сверху над списком) → **Add project from disk**.
3. В диалоге выберите **корневую папку проекта**: внутри неё должны лежать папки **`Assets`**, **`Packages`**, **`ProjectSettings`** рядом с этим `README.md`.  
   Пример пути на Windows: `C:\Work\Git\FPS\FPS Shooter` — не выбирайте только вложенную `Assets`, нужна **вся** папка репозитория.
4. Проект появится в списке. **Дважды щёлкните** по строке проекта — откроется редактор с привязанной версией Unity (Hub подставит **6000.3.11f1**, если она установлена).

### Первое открытие редактора

1. Окно может предложить **обновить/project upgrade** — для учебного репозитория безопаснее открывать **той же версией**, что в `ProjectVersion.txt`, а не соглашаться на апгрейд в другую major-версию без необходимости.
2. Внизу Unity идёт полоса **Importing / Compiling** — дождитесь тишины в консоли (окно **Console** можно открыть через **Window → General → Console**).
3. Откройте сцену: в **Project** найдите `Assets/Scenes/SampleScene` и дважды щёлкните по `SampleScene`.

Скриншоты в репозиторий не вложены: подписи кнопок в Hub иногда меняют от релиза к релизу; ориентируйтесь на английские названия выше. Если что-то не находится — в Hub есть **поиск** по настройкам или меню **Help**.

## Что значит «запустить проект»

| Действие | Что происходит |
|----------|----------------|
| **Play в редакторе** | Стартует текущая открытая сцена прямо в Unity. Это основной способ проверки геймплея во время разработки. |
| **File → Build and Run** | Собирается исполнимый файл (`.exe` на Windows и т.д.) — нужно, когда хотите отдать игру без Unity. Для обучения и отладки чаще достаточно **Play**. |

Этот репозиторий — **исходники Unity-проекта**, а не готовый `.exe`. Запуск всегда через установленный **Unity Editor** (или через собранный билд после настройки сцены).

## Где лежит код и что уже сделано

- Скрипты: `Assets/Scripts/Game/...`
- В проекте включены, в частности: **Input System**, **AI Navigation**, **URP** (см. `Packages/manifest.json`).

### Реализовано в коде

- Команды и урон: `CombatTeam`, `TeamMember`, `Health` (дружественный огонь отключён).
- Стрельба: `HitscanWeapon` — луч, выбор ближайшего валидного `Health`, сброс кулдауна между раундами.
- Управление: `FpsMotor`, `FpsLook`, `FpsInputDriver` — читает карту **Player** из `InputSystem_Actions` (поле `actions` в инспекторе).
- Раунды: `RoundOutcomeResolver` (вылет / таймер с суммой HP), `RoundManager` (счёт, пауза между раундами, респавн через `SpawnPoint`).
- HUD: `HudController` (три `UnityEngine.UI.Text`: таймер, счёт, баннер).
- Боты: `FpsMobController` — `NavMeshAgent`, поиск противника команды, наведение, `TryManualFire` в пределах дистанции.
- Редактор: меню **FPS Shooter → Drop Empty Round Manager** — создаёт пустой объект с `RoundManager` (остальные ссылки в инспекторе вешаются вручную).

## Словарь терминов (если пункты ниже непонятны)

- **Сцена (Scene)** — один «уровень»: список объектов в иерархии слева. Файлы сцен лежат обычно в `Assets/...` и имеют расширение `.unity`.
- **Иерархия (Hierarchy)** — дерево объектов на сцене.
- **Инспектор (Inspector)** — панель справа: свойства выбранного объекта и его компонентов.
- **Компонент** — кусок логики на объекте (например `CharacterController`, `Health`). Добавляются кнопкой **Add Component**.
- **Prefab** — сохранённый шаблон объекта; здесь основной акцент на ручной сборке в сцене.
- **Static** — у Mesh Renderer / объекта: отметка, что объект **не двигается**. Так пол проще участвует в запекании навигации.
- **NavMesh (навигационная сетка)** — «зона, по которой могут ходить агенты» (боты). Её создают командой **Bake** в окне Navigation — без этого `NavMeshAgent` у ботов не сможет найти путь.
- **Bake** — предрасчёт данных (здесь — сетка ходьбы по полу и препятствиям).

## Быстрый задел в редакторе

1. Откройте нужную сцену (например `Assets/Scenes/SampleScene.unity`, если она есть).
2. Меню **FPS Shooter → Drop Empty Round Manager** — в сцене появится объект с компонентом `RoundManager`.
3. Дальше по списку ниже добавьте пол, игрока, ботов, точки спавна, HUD и **перетащите ссылки** в `RoundManager` (он без них не знает, кого спавнить и что показывать).

## Что нужно собрать в сцене (вручную), по шагам

### 1. Пол

- Создайте **куб** или **Plane** (меню **GameObject → 3D Object**).
- Растяните по размеру арены при необходимости.
- Включите у него флаг **Static** (в инспекторе сверху), чтобы пол участвовал в навигации как неподвижная геометрия.
- При необходимости в окне **Navigation** отметьте область как **Walkable** (в зависимости от настроек агента и типа коллайдера).

### 2. Запекание NavMesh

- Откройте **Window → AI → Navigation** (или оверлей Navigation в сцене).
- Нажмите **Bake** по полу.

Без запечённой сетки боты с `NavMeshAgent` **не будут двигаться**.

### 3. Игрок

На одном объекте (капсула удобна как старт):

- `CharacterController`
- `TeamMember` (команда **ALPHA**)
- `Health`
- `FpsMotor`, `FpsLook`  
  - у `FpsLook`: «тело» — корень персонажа; **pitchPivot** — дочерний объект уровня головы (высота примерно **1.55–1.6** от ног).
- Дочерний объект с камерой: `Camera`, `AudioListener`, `HitscanWeapon`; в `HitscanWeapon` поле **wielderTeam** укажите на тот же `TeamMember`, что на теле.

### 4. Ввод

- На тело игрока: `FpsInputDriver`.
- Поле **actions**: назначьте ассет `Assets/InputSystem_Actions` (или ваш `.inputactions`, если переименовали).
- Ссылки: **motor**, **look**, **weapon** (камера / объект с `HitscanWeapon`).

### 5. Боты

Для каждого бота (капсула):

- `CapsuleCollider`, `NavMeshAgent`
- `TeamMember` (**ALPHA** или **BETA**)
- `Health`
- Как у игрока: дочерний узел с `HitscanWeapon` и привязкой `wielderTeam`
- `FpsMobController`: **affiliation**, **primary** (оружие) и прочие поля по инспектору

### 6. Точки спавна

- Создайте пустые **Empty** объекты на уровне пола.
- На каждый повесьте `SpawnPoint`.
- Минимум **4 точки**: хотя бы **2 на ALPHA** и **2 на BETA** (или больше — по задумке матча).

### 7. HUD

- **Canvas** + три объекта с **Text** (старый `UnityEngine.UI.Text`, как ожидает `HudController`).
- Пустой объект с `HudController`; в инспекторе привяжите три текстовых поля (таймер, счёт, баннер). При необходимости вызовите привязку из скрипта, если есть **BindOptional**.

### 8. RoundManager

На объекте с `RoundManager` (в том числе после **Drop Empty Round Manager**):

- Перетащите ссылку на **HUD** (объект с `HudController`).
- Массив **всех** `Health`, которые участвуют в бою (игроки и боты).
- Массив всех `SpawnPoint`.
- Параметры: длительность раунда, пауза между раундами, число **побед в матче** (например **4**).

## Типичные проблемы при первом открытии

| Симптом | Что проверить |
|--------|----------------|
| Hub пишет, что **нет подходящего редактора** / просит другую версию | Установите **6000.3.11f1** (раздел [Версия Unity](#версия-unity)). В списке проектов у строки проекта можно нажать на версию и выбрать установленный редактор. |
| Редактор открылся в **Safe Mode** (безопасный режим) | Откройте **Console**: там красные ошибки компиляции. Часто — другая версия Unity или битый импорт. Попробуйте **Assets → Reimport All** после исправления версии. |
| Долгий «зависон» при первом запуске | Первый импорт и компиляция могут занять **10–30+ минут** на слабых дисках — это нормально. Не закрывайте Unity до окончания полосы прогресса. |
| Вопрос про **Input Manager** vs **Input System** | Этот проект рассчитан на **новый Input System**. В **Edit → Project Settings → Player → Other Settings → Active Input Handling** должно быть **Input System Package** (или **Both**). Если поставить только старый Input Manager, управление из `FpsInputDriver` не заработает. |
| Вся графика **фиолетовая** / шейдеры не те | Проект на **URP**. В **Project Settings → Graphics** и **Quality** должны быть назначены URP-ассеты из проекта (часто лечится открытием стандартного URP-туториала ассета или повторным импортом `Settings`, если вы меняли ветку git). |
| **Play** нажат, сцена пустая / ничего не происходит | См. раздел [Что нужно собрать в сцене](#что-нужно-собрать-в-сцене-вручную-по-шагам): в `SampleScene` может не быть игрока и `RoundManager`. |
| Боты **стоят на месте** | Не сделан **Bake** NavMesh по полу или пол не **Static** / не входит в зону навигации (см. шаги 1–2 в сборке сцены). |
| Ошибки **Git** / папка без части файлов | Клонируйте репозиторий целиком (`git clone`). Если в будущем подключат **Git LFS**, понадобится установленный `git-lfs` и `git lfs pull`. |

Если после шагов выше ошибка повторяется, скопируйте **первую красную строку** из **Console** (полный текст) — по ней быстрее найти причину.

## Итог

Пока **SampleScene** не содержит этих объектов и ссылок, проект **открывается и компилируется**, но **игровой цикл не собран** — это нормально для репозитория с готовым кодом и ручной сценой. Соберите сцену по шагам выше, затем нажмите **Play**.

Если нужно, в следующей итерации можно добавить редакторский «генератор сцены» (пол + игрок + боты + HUD одной кнопкой) поверх этой базы.
